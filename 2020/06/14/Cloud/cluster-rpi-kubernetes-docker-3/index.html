1.3.2 Docker Installation

Run the following 4 commands one by one:

  • curl -sSL https://get.docker.com | sh
  • sudo systemctl enable docker
  • sudo systemctl start docker
  • sudo usermod -aG docker pi

Afterwards, reboot all [Raspberry Pi].

1.3.3.1 Docker Pull, Docker Run, Docker Exec

Due to respective architectures of these 4 [Raspberry Pi]s, I tried to use [Docker Hub] image [arm64v8/alpine] for pi01, and [Docker Hub] image [arm32v6/alpine] for pi02, pi03 and pi04. However, it finally turns out that [Pi4 64-bit raspbian kernel] is NOT quite stable yet.

pi01

It looks [Docker Hub] images from repository arm64v8 are NOT working properly on my [Raspberry Pi 4 Model B Rev 1.4 8GB] with [Pi4 64-bit raspbian kernel]. To further demonstrate this, I’m trying some other images from repository either arm64v8 or aarch64 (deprecated):

None of the three images [arm64v8/alpine], [arm64v8/ubuntu], [aarch64/ubuntu] can UP for 3 seconds.

😓 😔

Well, I can pull alpine and keep it UP instead of arm64v8/alpine though.

However, due to a bunch of the other unstabilities, I downgrade the system from [Pi4 64-bit raspbian kernel] to [Raspberry Pi OS (32-bit) with desktop 2020-05-27]. And it’s interesting that image from arm32v6/alpine are exactly the same as image directly from alpine, namely 3ddac682c5b6.
