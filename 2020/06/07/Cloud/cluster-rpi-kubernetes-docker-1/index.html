3. Raspberry Pi Cluster Configuration

This section partly refers to Build a Raspberry Pi cluster computer, Installing MPI for Python on a Raspberry Pi Cluster and Instructables: How to Make a Raspberry Pi SuperComputer!.

Actually, the cluster can certainly be arbitrarily configured as you wish. A typical configuration is 1-master-3-workers, but which one should be the master? Is it really a good idea to ALWAYS designate the MOST powerful one as the master? Particularly, in my case, 4 Raspberry Pis are of different versions, so that they are of different computing capability.

3.1 Configure Hostfile

It’s always a good idea to configure hostfile on the master node. However, as reasons mentioned above, there is NO priority among ALL nodes in my case, I configured hostfile for ALL 4 Raspberry Pis.

3.2 SSH-KEYGEN

In order to test multiple nodes across the cluster, we need to generate SSH keys to avoid inputting password for logging into the other nodes all the time. In such, for each Raspberry Pi, you’ll have to generate a SSH key by ssh-keygen -t rsa, and push this generated key using command ssh-copy-id onto the other 3 Raspberry Pis. Finally, for a cluster of 4 Raspberry Pis, there are 3 authorized keys stored in file /home/pi/.ssh/authorized_keys on each of the 4 Raspberry Pis.
