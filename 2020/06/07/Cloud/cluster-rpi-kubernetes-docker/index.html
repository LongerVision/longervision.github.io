3. Raspberry Pi Cluster Configuration

This section is partly referred to Build a Raspberry Pi cluster computer, Installing MPI for Python on a Raspberry Pi Cluster and Instructables: How to Make a Raspberry Pi SuperComputer!.

Actually, the cluster can certainly be arbitrarily configured as you wish. A typical configuration is 1-master-3-slaves, but which one should be the master? Is it really a good idea to ALWAYS designate the MOST powerful one as the master? Particularly, in my case, 4 Raspberry Pis are of different versions, so that they are of different computing capability.

3.1 Configure Machinefile

3.2.2 Test mpi4py helloworld.py

3.3 SSH-KEYGEN

In order to be able to test any configurations, we need to generate SSH keys to avoid logging with passwords all the time. For each Raspberry Pi, you’ll have to generate a SSH key by ssh-keygen -t rsa, and push this generated key using command ssh-copy-id 3 times. Finally, for a cluster of 4 Raspberry Pis, in each file /home/pi/.ssh/authorized_keys, there are 3 authorized keys stored.

4.3.2 Docker Installation

Run the following 4 commands one by one:

  • curl -sSL https://get.docker.com | sh
  • sudo systemctl enable docker
  • sudo systemctl start docker
  • sudo usermod -aG docker pi

Afterwards, reboot all Raspberry Pi.

5. Kubernetes

Kubernetes (commonly stylized as k8s) is an open-source container-orchestration system for automating application deployment, scaling, and management. (cited from Wikipedia)

On Raspberry Pi, a lightweight variant of Kubernetes is normally preferred. A variety of choices are available:

  • MicroK8s: MicroK8s is only available for 64-bit Ubuntu images. (Cited from How to build a Raspberry Pi Kubernetes cluster using MicroK8s: Setting up each Pi)
  • k3s, k3d, k3d
  • Minikube
  • kind
  • Kubeadm
